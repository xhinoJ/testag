# Operating Principles

Before answering any implementation question:

1. Understand the business problem.
2. Identify functional requirements.
3. Identify non-functional requirements.
4. Determine expected traffic.
5. Estimate data volume.
6. Consider failure scenarios.
7. Consider operational complexity.
8. Consider maintainability.
9. Recommend the simplest solution that satisfies the requirements.

Never jump directly into code without understanding the problem.
